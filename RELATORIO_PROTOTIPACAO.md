# Relatório de Prototipação - Alerta Fumaça

## Sistema Utilizado

Para a prototipação do aplicativo **Alerta Fumaça**, foi utilizada uma abordagem de **prototipação funcional** usando tecnologias web nativas:

- **HTML5**: Estruturação das telas e componentes da interface
- **CSS3**: Estilização, design responsivo e acessibilidade (alto contraste)
- **JavaScript (ES6+)**: Lógica de interação, navegação e persistência de dados

### Justificativa da Escolha

A escolha por HTML/CSS/JavaScript foi feita pelos seguintes motivos:

1. **Interatividade Real**: Diferente de ferramentas de mockup estático (como Balsamiq ou Figma), o protótipo desenvolvido é totalmente funcional e interativo, permitindo que os avaliadores experimentem todas as funcionalidades como se fosse o sistema final.

2. **Acessibilidade**: O protótipo pode ser acessado diretamente no navegador, sem necessidade de instalação de software adicional ou contas em plataformas específicas.

4. **Persistência de Dados**: Utilizando localStorage, é possível simular o comportamento real do sistema, com dados sendo salvos entre sessões.

5. **Notificações do Sistema**: A implementação permite testar funcionalidades avançadas como notificações do navegador, que não seriam possíveis em ferramentas de mockup tradicionais.

## Avaliação da Ferramenta

### Vantagens

-  **Interatividade Completa**: Todos os botões, formulários e navegação funcionam como no sistema real
-  **Sem Dependências Externas**: Não requer contas ou licenças de software
-  **Acesso Universal**: Funciona em qualquer navegador moderno
-  **Performance**: Carregamento instantâneo, sem necessidade de servidor
-  **Customização Total**: Controle completo sobre cada aspecto da interface e comportamento
-  **Teste de Funcionalidades Reais**: Notificações, localStorage, validações, etc.

### Desvantagens

-  **Tempo de Desenvolvimento**: Requer mais tempo do que ferramentas de mockup rápidas
-  **Visualização de Fluxos**: Ferramentas como Figma facilitam a visualização de fluxos de navegação

### Conclusão

A escolha por HTML/CSS/JavaScript foi adequada para este projeto, pois permitiu criar um protótipo de alta fidelidade que demonstra não apenas a aparência, mas também o comportamento real do sistema. Isso é especialmente importante para um aplicativo focado em acessibilidade e usabilidade, onde a experiência de interação é crucial para a validação do design.

O protótipo desenvolvido serve tanto como demonstração visual quanto como base técnica para a implementação final do sistema.
